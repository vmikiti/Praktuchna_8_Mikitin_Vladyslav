## 2. Відповіді на завдання для звіту

### 2.1. Що таке незмінність рядків (immutability) у C#?
У C# рядки (`string`) є **незмінними**. Це означає, що після створення об'єкта в пам'яті його вміст не може бути змінений. Будь-яка модифікація (наприклад, конкатенація `+` або `Replace`) насправді створює новий рядок, а старий залишається в пам'яті до моменту очищення збирачем сміття (Garbage Collector).
* **Проблеми:** При великій кількості ітерацій (наприклад, у циклах) це призводить до значних витрат пам'яті та сповільнення роботи програми.
* **Рішення StringBuilder:** `StringBuilder` використовує внутрішній масив символів (буфер), який можна змінювати напряму без створення нових об'єктів.

### 2.2. Застосування StringBuilder у програмі
Я застосував `StringBuilder` у наступних місцях:
1. **`Student.GetFormattedInfo`**: для динамічної побудови картки студента з графічними межами.
2. **`TextProcessor.BuildGroupReport`**: для генерації табличного звіту групи, де важливо зберігати структуру при великій кількості даних.
3. **`AdvancedLogger`**: для накопичення повідомлень логування перед їх виводом або збереженням.
**Чому саме там?** У цих методах відбувається послідовне додавання багатьох фрагментів тексту. Звичайна конкатенація створювала б сотні тимчасових об'єктів, що неефективно.

### 2.3. Порівняння продуктивності (ComparePerformance)
Результати тестування на моїй системі:
* **1 000 ітерацій:** Різниця майже непомітна (String: ~1ms, StringBuilder: <1ms).
* **10 000 ітерацій:** String — ~140-160ms, StringBuilder — <1ms.
* **100 000 ітерацій:** String — ~6000+ms, StringBuilder — ~2-3ms.
**Висновок:** При великих обсягах даних `StringBuilder` швидший у тисячі разів.

### 2.4. Методи класу String у коді
Найбільш вживані методи:
* **`Split()`**: Використовується для перевірки ПІБ та розбиття вхідних даних при імпорті.
* **`Join()`**: Для формування рядків формату CSV.
* **`Contains()`**: Для пошуку ключових слів у нотатках студентів.
* **`Trim()` / `ToLower()` / `ToUpper()`**: Для нормалізації введених даних та порівняння без урахування регістру.

### 2.5. Реалізація IsPalindrome та Normalize
* **`Normalize(text)`**: Рядок розбивається на слова за допомогою `Split(..., RemoveEmptyEntries)`, що видаляє зайві пробіли, а потім з'єднується через `Join` з одним пробілом.
* **`IsPalindrome(text)`**: Текст очищується від пробілів, переводиться в один регістр і порівнюється зі своєю реверсною копією (`text.Reverse()`).

## 3. Контрольні питання

1. **Відмінність string від StringBuilder:** `string` — незмінний, `StringBuilder` — динамічний буфер.
2. **Пам'ять при конкатенації:** Відбувається масове створення тимчасових об'єктів у купі (Heap), що змушує Garbage Collector працювати інтенсивніше.
3. **Методи:** `IndexOf/LastIndexOf` — пошук позиції; `Split/Join` — робота з масивами рядків; `Trim` — очищення від пробілів.
4. **Інтерполяція ($"")**: Читабельніша та легша в написанні, ніж `string.Format`.
5. **StringComparison.OrdinalIgnoreCase**: Дозволяє порівнювати рядки без урахування регістру найшвидшим способом.
6. **Оператор +**: Компілятор перетворює його на `string.Concat()`.
