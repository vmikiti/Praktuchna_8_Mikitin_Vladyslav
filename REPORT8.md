1. Класи для роботи з файловою системою
Для роботи з файлами та папками в C# (простір імен System.IO) я використовував:
File/Directory: Статичні класи для швидких операцій (створення, копіювання, видалення).
FileInfo/DirectoryInfo: Об'єктно-орієнтовані класи для отримання детальних даних (розмір файлу, час створення).
Path: Для безпечного об'єднання шляхів через Path.Combine.
StreamReader/StreamWriter: Для текстового читання та запису через потоки.

2. Різниця між File/FileInfo та Directory/DirectoryInfo
File / Directory: Статичні класи. Кожен метод вимагає передачі шляху. Кращі для одноразових операцій.
FileInfo / DirectoryInfo: Екземплярні класи. Створюється об'єкт для конкретного файлу/папки. Зручні для багаторазових запитів до одного об'єкта (наприклад, у моєму методі CleanOldBackups ми використовували FileInfo, щоб перевірити дату створення кожного файлу).

3. Робота StreamReader та StreamWriter
Це обгортки над потоками, які працюють з текстом.
StreamWriter: Використовується для запису. Я застосував його в методі LogAction (Варіант 1) та для експорту в CSV.
StreamReader: Для читання великих обсягів тексту.
Коли використовувати: Коли потрібно записувати дані поступово (дозапис логів) або працювати з текстовими файлами, які можуть бути великими за розміром.

4. Серіалізація через System.Text.Json
Це процес перетворення об'єкта StudentGroup у текстовий рядок JSON для збереження на диску.
Як працює: Метод JsonSerializer.Serialize аналізує публічні властивості об'єкта та формує дерево ключ-значення.
Атрибути: Я використав [JsonIgnore] для розрахункових полів (наприклад, AverageGrade), щоб не захаращувати файл даними, які можна обчислити динамічно.

5. Винятки та їх обробка
При роботі з файлами виникали:
FileNotFoundException (файл видалено).
UnauthorizedAccessException (файл відкритий в іншій програмі).
IOException (помилки диска).
Обробка: Використано блоки try-catch у FileManager. Також створено власний виняток InvalidFileFormatException для випадків, коли JSON-файл пошкоджений.

6. Реалізація резервного копіювання
Бекап реалізовано у методі CreateBackup:
Отримання імені файлу через Path.GetFileNameWithoutExtension.
Додавання часової мітки (DateTime.Now).
Копіювання файлу в папку Backups за допомогою File.Copy.
Впроваджено систему очищення (CleanOldBackups), яка видаляє копії, старіші за вказану кількість днів.

7. Труднощі та нові знання
Труднощі: Конфлікти імен при оголошенні екземпляра FileManager та обробка кирилиці в JSON.
Нове: Навчився працювати з ротацією логів (перевірка розміру файлу перед дозаписом), використовувати using var для автоматичного звільнення ресурсів та налаштовувати JsonSerializerOptions.

Контрольні питання

1. File.WriteAllText() vs StreamWriter: WriteAllText зручний для дрібних файлів (відкриває-записує-закриває за один крок). StreamWriter кращий для великих обсягів або дозапису (Append), бо дозволяє тримати потік відкритим.

2. Серіалізація/Десеріалізація: Серіалізація — об'єкт → рядок. Десеріалізація — рядок → об'єкт.

3. Навіщо using? Гарантує виклик Dispose() (закриття файлу) навіть якщо виникне помилка. Без нього файл може залишитися «зайнятим» системою.

4. Помилки доступу: Обробляються через catch (UnauthorizedAccessException). Користувачеві виводиться порада закрити файл в інших програмах.

5. JsonSerializerOptions: Потрібен для налаштування формату (відступи, ігнорування регістру, кодування). Без нього JSON записується одним довгим нечитабельним рядком.

6. Небезпека відносних шляхів: Програма може шукати файли в різних папках залежно від того, як її запустили (через термінал, VS або .exe). Краще використовувати AppDomain.CurrentDomain.BaseDirectory або Path.Combine.

7. Бінарні vs Текстові файли: Текстові (JSON, CSV) — легко читати людині та редагувати. Бінарні — швидші, займають менше місця та захищають дані від легкого перегляду.
